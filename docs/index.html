§ 03 the actual pain

Open chat with one model
doesn't ship decisions.

You've spent three hours with Claude on a single architecture call. The doc is longer. The verdict is no closer. You're not stuck because the model isn't smart — you're stuck because nothing in the loop is forcing convergence, and there's no one in the room willing to disagree with you.

err · sycophancy_loop

Your AI agrees with everything.

Single-model sessions become sycophancy spirals. There's no critic, no skeptic, no contrarian — just one model nodding along to whatever you said last. The more you prompt, the more it confirms what you already thought.

warn · no_termination

Re-prompting is the new doom-scroll.

Hours of "make it better," "try again," "go deeper." No structure, no closure — an infinite refinement loop with no termination condition built in.

fail · no_verdict

You walk away with vibes, not a verdict.

Open-ended chat has no forcing function. The thread sprawls, the doc bloats, and you're left convinced something in there must be the answer. It isn't.

§ 04 the method

Pick a mode.
Run the phases.
Get a verdict.

Forge orchestrates agents through a deterministic state machine. Each phase enforces its own focus, message limits, and loop detection — so deliberations converge instead of spiraling.

i.

Pick your mode.

Choose from eleven deliberation workflows — copywriting, idea validation, ideation, feasibility, site survey, business plan, GTM, VC pitch, technical review, red team — or write your own.

ii.

Define the question.

Set your goal. Select which archetypes participate. Configure round limits to prevent runaway discussion.

iii.

Watch structured phases.

The session moves automatically through Discovery → Research → Synthesis → Drafting → Verdict. No one gets stuck on framing forever.

iv.

Get forced closure.

Loop detection and message limits prevent circular debate. Agents are pushed through evidence gathering to consensus.

v.

Export the result.

Full transcript, final draft, per-section decisions. Markdown, JSON, or HTML.

§ 07 what's inside

Built for decisions
that actually ship.

i. the ensemble

Multi-archetype deliberation.

Five culture-neutral reasoning archetypes — skeptic, pragmatist, analyst, advocate, contrarian — each bringing a distinct stance. No made-up names, no personalities, just roles.

ii. the scribe

Grounded in your real code.

When the debate needs facts, one of the archetypes pauses to read your actual source files — not a summary, the files — and comes back with citations. Decisions grounded in what you shipped, not what someone remembered.

iii. the machine

Every session ends with a verdict.

Sessions run through a fixed arc: Discovery → Research → Synthesis → Drafting → Verdict. No spinning forever on framing. No session without an exit door.

iv. the key

No new subscription. No new key.

If you can already run Claude Code, Forge works. Your existing auth carries through — no paste-in API key, no vendor portal, no extra monthly line item.

v. the bell

Circular debate gets interrupted.

When agents start repeating themselves, the session notices and forces the next phase. Repeat questions get capped. You always walk away with a result — not a half-finished thread.

vi. the chamber

Watch it happen in your terminal.

Phases, turns, who's speaking, who's being disputed — all live, in place. No dashboards to open, no browser tabs to juggle, no heavyweight desktop app. Just the deliberation, rendered.

vii. the console

Live operator control.

Press a to open Agent Control — swap each agent's model (Claude / Gemini / more coming), toggle skills with k, pause a loud voice, or force-speak a specific perspective. Every change applies to the next turn without restarting the session.

viii. the archive

Every session is a self-describing artifact.

Each run produces a workdir with per-agent message logs, a consensus directory that auto-captures anything tagged [CONSENSUS] or [SYNTHESIS], and the resolved skill bundle each agent was primed with. Audit the decision, diff it against another run, or hand it to the next reviewer.

§ 09 frequently asked

Stuff people ask
first.

If it's not answered here, the answer is probably in the repo. Otherwise open an issue — we read them.

How is this different from Claude or GPT directly?

A single model agrees with whatever you said last. Forge runs five archetypes who don't through a fixed workflow — Will It Work (define → evidence → debate → verdict), VC Pitch (pitch → market → unit-economics → debate → memo), Red Team (recon → threat-model → attack-chains → mitigations), or any of eight others. You don't get a nicer draft — you get a real decision, with the disagreement visible and the verdict attributable.

Can I run different models for different agents?

Yes. Press a in the running TUI to open the Agent Control panel. From there you can assign each agent its own provider and model — e.g. run the Skeptic on Claude Opus, the Pragmatist on Gemini 2.5 Flash, and the Analyst on GPT-4o. Changes apply to the very next response, no restart. Anthropic ships always-on; Gemini activates when GEMINI_API_KEY is set; OpenAI activates when OPENAI_API_KEY is set.

Can I bring my own skills?

Yes. Drop markdown files in skills/ (per-agent, per-mode, or shared), or ship a skills.sh in the project root that populates them at session start. Press k on an agent in the control panel to browse the full catalog and toggle individual skills live. User-level skills at ~/.claude/skills/forge/*.md are also discovered automatically.

What does it cost to run?

Forge itself is free. Inference costs pass through to whichever model provider you route each agent to. If you already pay for Claude Pro/Max your Anthropic calls inherit that; Gemini billing goes to Google. The TUI shows a per-session cost meter so you know order-of-magnitude before it matters.

Does it work offline?

The deliberation engine is local, but the models aren't. A self-hosted provider (Ollama, LM Studio, vLLM) would drop in as another IProvider — scoped for a later phase but the interface is ready.

Do I need an API key?

No. If you can already run claude in your terminal, Forge works — your existing Claude Code auth carries through. No pasted API key, no new vendor portal, no extra monthly bill. Your session is your session.

Is the debate deterministic?

The structure is deterministic — phases run in order, message limits are enforced, and research requests are serialized. The content of each turn is not: different runs will surface different evidence and arguments, but the shape of the deliberation is always the same.

Can I use my own models?

Yes. The agent runner is a swappable interface — Claude Code is the default, but you can wire in GPT, Gemini, a local Llama, or anything else that can answer a prompt. The deliberation engine doesn't care which model is behind each archetype.

How do I extend it with new modes?

Every mode is a small config object: phase sequence, per-phase focus, success criteria, required sections. Copy an existing one, change the shape, register it, and it shows up in the CLI — no changes to the core engine. Custom modes ship alongside the built-in ten.
