CH. 01 the honest framing

CivicVote doesn't exist yet.
That's kind of the point.

editor's note

A product is the only honest way to show what a deliberation engine does.

Feature lists lie. Demo videos in isolation lie. So we pointed Forge at one made-up-but-plausible product — a civic voting platform for towns the size of Beer Sheva or Setúbal — and ran every mode end-to-end.

The sessions below are real. The verdicts are real. The product isn't shipping. But the shape of the work would look like this if it were yours.

the subject

CivicVote — decentralized local-municipal voting.

Pitch Restore the true public-local narrative — local decisions decided locally, not captured by national political feeds.
Identity DID-based (did:key). No phone numbers. Coercion-resistant, spam-resistant.
Infra Local-first — runs on the municipality's servers, not a global SaaS.
Tallies Verifiable — every voter can confirm their vote was counted without revealing how they voted.
The tension Today, zoning fights get hijacked by Twitter. Council meetings collapse to 5% turnout. CivicVote is the missing layer.

The Skeptic killed the first two pitches.

CH. 03 can it survive contact with reality?

A verdict is easier
when five archetypes disagree first.

The validation verdict said GO-with-caveats. So the team runs the two modes that usually kill new ventures: will-it-work for feasibility — can we get 30% turnout in a 50k-person pilot? — and red-team for the adversaries nobody wants to think about until launch night.

MAYBE-IF:
pilot city under 100k.
forge -m will-it-work ~11 min · forced verdict

Will 30% of a town actually turn out? MAYBE-IF

No fence-sitting. No "it depends." The engine forces a YES / NO / MAYBE-IF with explicit conditions. Pilot constrained to towns where a civic-engagement baseline already exists.

define → evidence → debate → verdict
Named threat: coercive spouse.
Mitigation: re-vote window.
forge -m red-team ~14 min · ranked mitigations

Who attacks this, and how?

Named adversaries: nation-state manipulation, local incumbents, coercive employers and spouses. Each adversary gets a concrete kill chain. Defender archetype scores attacks against real detection capacity. Output: a ranked mitigation plan, not a risk matrix.

recon → threat-model → attack-chains → mitigations
CH. 04 the code exists. is it any good?

Four specialists audit
the voting protocol.

Code is written. Ballot encryption, tally verification, DID registration — it all runs. Before a single resident touches it, the team runs tech-review: architect reads module boundaries, perf engineer hunts hot paths, security reviewer traces untrusted input, test engineer asks what regression would actually ship. Output is a findings report with file paths and severity — not vibes.

High-sev finding:
nonce re-use in ballot commit.
Each finding has a file path.
Not hand-waved.
forge -m tech-review ~18 min · findings by severity

Architect, perf, security, tests — all in one pass.

One of the archetypes pauses the debate to read your actual source files — not a summary, the files — and comes back with citations. Decisions grounded in what you shipped, not what someone remembered.

recon → architecture-read → hotspot-dive → report

CH. 07 you're not just watching the debate

CH. 08 the engine, stripped down

Five archetypes.
A phase machine. A verdict.

Three hours re-prompting one model gets you a longer draft, not a decision. Forge forces convergence by running five culture-neutral reasoning archetypes through a deterministic phase machine. The content of each turn is free; the structure is not.

the ensemble
Skeptic Demands evidence. Kills arguments that lean on vibes. Named the coercive-spouse threat model.
Pragmatist Forces closure. "What's the actual verdict." Shut down the five-message digression on ZK proofs.
Analyst Traces second-order effects. Found that per-referendum pricing kills civil-society use cases.
Advocate Holds the thesis. Defends the vision when the Skeptic is about to talk the team out of shipping.
Contrarian Asks the uncomfortable question. "What if low turnout is the feature, not the bug?"
the phase machine

Every mode is a config: phase sequence, per-phase focus, success criteria, required outputs. The engine enforces them. When a phase satisfies its criteria, it advances. When agents start repeating themselves, the next phase triggers automatically. You always walk away with a result — not a half-finished thread.

canonical arc
discovery → research → synthesis → drafting → verdict
audit trail per session
per-agent message logs · consensus directory · resolved skill bundle
