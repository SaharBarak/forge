The problem

Decisions die in the gap between opinion and action.

Your team burns cycles on decisions that should take minutes, not meetings. Product choices die in endless Slack threads while engineers debate architecture in circles. Critical feasibility questions get answered by whoever speaks loudest, not who has the best evidence.

Ideas die in Slack threads

No forcing function to reach actual conclusions. The conversation trails off and nobody remembers what was decided.

Meetings circle without deciding

Discussion without structure becomes performance, not progress. Three hours in, you're further from a decision than when you started.

Volume beats evidence

Seniority and loudness override the people who actually did the analysis. Decisions get made by whoever stops the meeting first.

How it works

Pick a mode. Run the phases. Get a verdict.

Forge orchestrates agents through a deterministic state machine. Each phase enforces its own focus, message limits, and loop detection — so deliberations converge instead of spiraling.

1

Pick your mode

Choose from eight deliberation workflows designed for different decision types — or write your own.

2

Define the question

Set your goal and select which agent archetypes participate. Configure round limits to prevent runaway discussion.

3

Watch structured phases

The ModeController enforces Discovery → Research → Synthesis → Drafting (or whatever the mode specifies). Phase transitions are automatic.

4

Get forced closure

Loop detection and message limits prevent circular debate. Agents are pushed through evidence gathering to a final consensus or verdict.

5

Export the result

Full deliberation transcript, final draft, per-section decisions. Markdown, JSON, or HTML.

Eight battle-tested modes

Each mode embeds a proven decision framework.

Modes aren't just prompts — they're full workflows with defined phases, message limits, success criteria, and loop detection tuned for that decision type.

The ModeController detects circular discussion and enforces phase transitions. Research requests are capped per topic. Sessions always terminate cleanly.

CLI-first Ink TUI

Real-time phase progression, agent stances, floor management, and consensus tracking in your terminal. No web dashboards, no Electron bloat.

Questions

Frequently asked.

How is this different from using Claude or GPT directly?

Forge structures the conversation through predefined deliberation modes with phase enforcement. Instead of open-ended prompting, you get workflows like Will It Work (define → evidence → debate → verdict) or Copywriting (discovery → research → ideation → synthesis → drafting). The ModeController prevents endless loops and forces closure — a 20-person engineering team choosing between monolith and microservices gets structured evidence gathering, formal debate, then a final vote.

Do I need an API key?

No direct Anthropic API key required. Forge shells out to your authenticated Claude Code CLI via @anthropic-ai/claude-agent-sdk. If you can run claude in your terminal, Forge inherits that authentication — no additional keys, no vendor-level lock-in.

Is the debate deterministic?

The structure is deterministic — phases run in order, message limits are enforced, and research requests are serialized. The content of each turn is not: different runs will surface different evidence and arguments, but the shape of the deliberation is always the same.

Can I use my own models?

Yes. The IAgentRunner interface lets you swap the backend. The default ClaudeCodeCLIRunner shells out to Claude Code, but you can wire in any other provider (including local models) that fits the interface. The phase executor doesn't care which model is behind it.

How do I extend it with new modes?

All deliberation modes live in src/lib/modes/index.ts with configurable phases, agent focus, success criteria, and loop detection. Copy an existing mode, change its phase list and requiredOutputs, register it, and the phase executor picks it up automatically. No changes needed to the core engine.
